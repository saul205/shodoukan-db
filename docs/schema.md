# Database schema

The SQLite database generated by the builder contains 15 tables organized into two groups: dictionary entries and kanji.

Arrays (readings, part-of-speech tags, etc.) are stored as JSON inside `TEXT` columns.

---

## Dictionary entries

These tables store JMDict data: words, readings, and definitions.

### `entries`

Root table. Each row represents one dictionary entry.

| Column | Type    | Description           |
|--------|---------|-----------------------|
| `id`   | INTEGER | Original JMDict ID    |

---

### `kanji_readings`

Kanji-written forms of an entry (`<k_ele>`).

| Column     | Type    | Description                                              |
|------------|---------|----------------------------------------------------------|
| `id`       | INTEGER | Primary key                                              |
| `entry_id` | INTEGER | Reference to `entries`                                   |
| `kanji`    | TEXT    | Kanji spelling, e.g. `食べる`                             |
| `priority` | TEXT    | JSON array of frequency tags, e.g. `["ichi1"]`           |
| `info`     | TEXT    | JSON array of notes, e.g. `["irregular kanji usage"]`    |

---

### `readings`

Kana readings of an entry (furigana, `<r_ele>`).

| Column     | Type    | Description                                              |
|------------|---------|----------------------------------------------------------|
| `id`       | INTEGER | Primary key                                              |
| `entry_id` | INTEGER | Reference to `entries`                                   |
| `text`     | TEXT    | Kana reading, e.g. `たべる`                               |
| `no_kanji` | INTEGER | `1` if the word is not normally written with kanji       |
| `priority` | TEXT    | JSON array of frequency tags                             |
| `info`     | TEXT    | JSON array of notes                                      |

---

### `reading_restrictions`

Links readings to the specific kanji spellings they apply to. A reading with no restrictions applies to all spellings of the entry.

| Column             | Type    | Description                        |
|--------------------|---------|------------------------------------|
| `reading_id`       | INTEGER | Reference to `readings`            |
| `kanji_reading_id` | INTEGER | Reference to `kanji_readings`      |

---

### `senses`

A sense groups definitions that share the same part of speech. One entry can have multiple senses (e.g. verb and noun).

| Column     | Type    | Description                                              |
|------------|---------|----------------------------------------------------------|
| `id`       | INTEGER | Primary key                                              |
| `entry_id` | INTEGER | Reference to `entries`                                   |
| `pos`      | TEXT    | JSON array of part-of-speech tags, e.g. `["noun"]`       |
| `misc`     | TEXT    | JSON array of miscellaneous notes                        |
| `dialects` | TEXT    | JSON array of dialects where this sense is used          |
| `info`     | TEXT    | JSON array of additional notes                           |

---

### `glosses`

Individual definitions of a sense, in English or other languages.

| Column     | Type    | Description                                     |
|------------|---------|-------------------------------------------------|
| `id`       | INTEGER | Primary key                                     |
| `sense_id` | INTEGER | Reference to `senses`                           |
| `text`     | TEXT    | Definition text, e.g. `"to eat"`                |
| `type`     | TEXT    | Semantic type, e.g. `"fig."` (figurative)       |
| `lang`     | TEXT    | Language code, e.g. `eng`, `dut`               |

**Full-text search:** the `glosses_fts` virtual table (FTS5) is kept in sync automatically via a trigger and enables efficient search by definition text.

---

### `cross_references`

References from a sense to another dictionary entry.

| Column      | Type    | Description                                        |
|-------------|---------|----------------------------------------------------|
| `id`        | INTEGER | Primary key                                        |
| `sense_id`  | INTEGER | Reference to `senses`                              |
| `reference` | TEXT    | Kanji or kana of the referenced entry              |
| `reading`   | TEXT    | Specific reading of the referenced entry           |
| `sense_idx` | INTEGER | Index of the referenced sense (when applicable)    |

---

### `examples`

Example sentences associated with a sense.

| Column        | Type    | Description                                    |
|---------------|---------|------------------------------------------------|
| `id`          | INTEGER | Primary key                                    |
| `sense_id`    | INTEGER | Reference to `senses`                          |
| `source_name` | TEXT    | Name of the source corpus, e.g. `tat`          |
| `source_id`   | TEXT    | Example ID in the source corpus (if available) |
| `text`        | TEXT    | Japanese example sentence                      |

---

### `example_sentences`

Translations of each example into different languages.

| Column       | Type    | Description                          |
|--------------|---------|--------------------------------------|
| `id`         | INTEGER | Primary key                          |
| `example_id` | INTEGER | Reference to `examples`              |
| `lang`       | TEXT    | Language code, e.g. `eng`            |
| `text`       | TEXT    | Translated sentence                  |

---

## Kanji

These tables store KANJIDIC2 data: individual characters with their readings and metadata.

### `kanji`

One row per kanji character.

| Column         | Type    | Description                                                    |
|----------------|---------|----------------------------------------------------------------|
| `literal`      | TEXT    | The kanji character (primary key), e.g. `食`                   |
| `grade`        | INTEGER | Japanese school grade (1–6, or 8 for general-use kanji)       |
| `stroke_count` | INTEGER | Number of strokes                                              |
| `freq`         | INTEGER | Frequency rank (lower = more common)                           |
| `jlpt`         | INTEGER | JLPT level (1–4)                                               |
| `on_readings`  | TEXT    | JSON array of on readings (Sino-Japanese), e.g. `["ショク"]`   |
| `kun_readings` | TEXT    | JSON array of kun readings (native Japanese), e.g. `["た.べる"]` |
| `nanori`       | TEXT    | JSON array of readings used in proper names                    |

---

### `kanji_meanings`

Meanings of a kanji in different languages.

| Column    | Type    | Description                              |
|-----------|---------|------------------------------------------|
| `id`      | INTEGER | Primary key                              |
| `literal` | TEXT    | Reference to `kanji`                     |
| `text`    | TEXT    | Meaning, e.g. `"eat"`, `"food"`          |
| `lang`    | TEXT    | Language code, e.g. `en`, `fr`, `es`    |

**Full-text search:** the `kanji_meanings_fts` virtual table (FTS5) enables search by kanji meaning.

---

## Relation index

### `entry_kanji`

Links each dictionary entry to the individual kanji characters that appear in its spellings. Built in the final step of the builder by scanning all `kanji_readings`.

| Column           | Type    | Description                                                |
|------------------|---------|------------------------------------------------------------|
| `entry_id`       | INTEGER | Reference to `entries`                                     |
| `literal`        | TEXT    | Individual kanji character                                 |
| `priority_score` | INTEGER | Frequency score of the entry (higher = more common)        |

This table makes it possible to find all words containing a given kanji, ordered by frequency. It is also the basis for generating hyperlinks between words and kanji in the UI.

**Scoring:** `ichi1`=1000, `ichi2`=500, `news1`=400, `spec1`=300, `gai1`=100. The `nf*` tags contribute between 4 and 200 points depending on their number.
